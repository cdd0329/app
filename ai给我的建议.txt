### 📱 模拟器配置流程

| 目标平台 | 所需前置工具 | 模拟器获取与启动方式 |
| :--- | :--- | :--- |
| **Android 模拟器** | [Android Studio](https://developer.android.com/studio) | **获取**：在 Android Studio 中，通过 **AVD Manager** 创建虚拟设备，需要下载一个系统镜像（推荐 x86_64 架构的版本）。<br>**启动**：可以在 AVD Manager 中点“播放”按钮启动，或使用命令 `flutter emulators --launch 模拟器名称`。 |
| **iOS 模拟器** (仅限 macOS) | [Xcode](https://developer.apple.com/xcode/) (需从 Mac App Store 下载，体量较大) | **获取**：安装 Xcode 后，打开 Xcode，进入 `Preferences > Components` 下载所需的 iOS 模拟器版本。<br>**启动**：在 VS Code 中，打开命令面板 (`Cmd+Shift+P`)，输入 **Flutter: Select Device** 并选择 iOS 模拟器，它会自动帮你启动。也可以手动通过 `open -a Simulator` 命令启动。 |

### 💡 完整环境配置参考

无论是 Android 还是 iOS 模拟器，都依赖于 Flutter 开发环境的完整搭建。一个非常权威的参考是 **Flutter 官方文档的《Set up and test drive Flutter》**，它是一份针对 VS Code 的快速配置指南。

它的核心流程大致如下：
1.  **安装 VS Code**，并确保系统已安装 **Git**。
2.  **在 VS Code 中安装 Flutter 插件**（这会同时安装 Dart 插件）。
3.  用 VS Code 命令面板 (`Cmd+Shift+P`) 选择 **Flutter: New Project**，根据提示让 VS Code 帮你**下载 Flutter SDK** 并自动添加到 PATH 环境变量。
4.  在 VS Code 命令面板运行 **Flutter: Run Flutter Doctor**，它会检查你的环境配置是否完整，并提示你修复任何缺失项。

### ⚙️ 各平台详细步骤参考

*   **Windows 用户**：除了通过 Android Studio 获取 Android 模拟器，还需要确保系统安装了 Git，并正确配置了 Flutter SDK 的环境变量（Path）。一个完整的 Windows 环境配置可以参考 GitHub 上的这份指南。
*   **macOS 用户**：如果你有 Android 和 iOS 双端开发的需求，需要同时安装 Android Studio 和 Xcode，这样就能同时获得 Android 和 iOS 的模拟器了。

是的，**需要下载**。这里的“Android 模拟器”是 Android Studio 这个大软件自带的一个核心功能，但它需要你在首次启动时联网下载必需的组件才能使用。简单来说，整个过程分两步：

### 📥 第一步：下载 Android Studio 本体

首先，你需要从官网下载并安装 Android Studio 这个“大软件”。它类似于一个专门用于开发安卓应用的“工作台”，你所有的工作都会在这里面完成。

### ⚙️ 第二步：在 Android Studio 里下载“模拟器”组件

装好 Android Studio 并首次启动时，它会引导你进行一个“设置向导”。关键就在这一步：

1.  **选择“自定义 (Custom)”安装**：在设置向导的“安装类型 (Install Type)”步骤，务必选择 **“Custom” (自定义)**。这能让你自己指定 Android SDK（开发工具包）的下载位置，防止它把好几个 G 的文件都塞进 C 盘。
2.  **下载核心组件**：在后续的“SDK 组件选择”页面，你需要勾选并下载几个关键组件，其中就包括 **Android Emulator (安卓模拟器)**。这里会下载几个 G 的系统文件，网络稳定很重要。
3.  **同意许可协议**：最后，需要同意所有相关的许可协议（License），然后 Android Studio 就会自动开始下载和安装你选中的所有组件了。

### 🚀 创建你的第一个虚拟手机

等所有组件都下载安装完，你就可以在 Android Studio 里创建属于你的虚拟手机了（也叫 AVD）：

1. 在 Android Studio 里找到 **“设备管理器” (Device Manager)**。
2. 点击 **“创建设备” (Create device)**，从列表里选一个你喜欢的手机型号（比如 Pixel 系列），然后点“下一步”。
3. 接下来，为这台虚拟手机选择一个操作系统版本（也就是之前下载的“系统映像”），选好后点“下一步”并根据需要调整配置，最后点“完成”就行。创建完成后，在设备管理器里点绿色的“播放”按钮，就能把这台虚拟手机“开机”了。

等你按这套流程走下来，就拥有了一个完整的 Flutter 安卓开发环境。`flutter doctor` 这个命令就是检查所有环境是否就绪的好帮手，其中就包括 Android 模拟器是否配置成功。
